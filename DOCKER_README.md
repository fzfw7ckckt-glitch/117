# OSINT Platform 2026 - Docker Edition

Complete containerized deployment of the hybrid OSINT platform with FastAPI, Celery, PostgreSQL, Redis, and Next.js frontend.

## 🎯 Quick Start (3 minutes)

```bash
# 1. Clone and enter directory
git clone <repo> osint-platform-2026
cd osint-platform-2026

# 2. Run setup script (auto-configures everything)
bash docker-setup.sh

# 3. Configure API keys
nano .env

# 4. Access services
# Frontend: http://localhost:3000
# API Docs: http://localhost:8000/docs
```

## 📦 What's Included

### Services
- **FastAPI Backend** - REST API with JWT auth, rate limiting
- **Celery Workers** - Distributed OSINT task processing (4 concurrent)
- **Next.js Frontend** - Modern React dashboard
- **PostgreSQL** - Primary database (15-alpine)
- **Redis** - Cache, message broker, result backend
- **Tor Proxy** - Anonymized connections
- **Nginx** - Production reverse proxy (optional)

### OSINT Tools Integrated
- Maigret (username search)
- Shodan (IoT/network reconnaissance)
- GeoSpy & Picarta (geolocation)
- YouControl (Ukrainian business intelligence)
- OpenSanctions (sanctions checking)
- SpiderFoot, Amass (passive recon)
- PhoneInfoga (phone lookup)
- And 150+ more in the tool catalog

## 🚀 Usage

### Basic Commands

```bash
# Start all services
docker-compose up -d

# View status
docker-compose ps

# Check logs
docker-compose logs -f api

# Stop all services
docker-compose down
```

### Using Makefile

```bash
make setup       # Initial setup
make up          # Start services
make logs        # View all logs
make logs-api    # View API logs
make shell-api   # SSH into API container
make health-check # Check service health
make clean       # Stop and remove everything
```

## 🔧 Configuration

### Environment Variables (.env)

Essential variables to set:

```bash
# Security
JWT_SECRET_KEY=<auto-generated or set manually>

# Database
DB_PASSWORD=<strong-password>

# API Keys (get from respective services)
SHODAN_API_KEY=
OPENAI_API_KEY=
GEOSPY_API_KEY=
YOUCONTROL_API_KEY=
```

### Multi-Worker Scaling

```bash
# Run 3 worker instances
docker-compose up -d --scale worker=3

# Verify
docker-compose ps | grep worker
```

### Resource Limits

Edit `docker-compose.yml`:

```yaml
worker:
  deploy:
    resources:
      limits:
        cpus: '2'
        memory: 2G
```

## 🐛 Troubleshooting

### Services won't start

```bash
# Check for port conflicts
lsof -i :8000
lsof -i :3000

# Check logs
docker-compose logs postgres
docker-compose logs redis

# Rebuild from scratch
docker-compose down -v
docker-compose build --no-cache
docker-compose up -d
```

### Database connection error

```bash
# Check PostgreSQL status
docker-compose exec postgres pg_isready -U osint

# View logs
docker-compose logs postgres

# Reset database
docker-compose down -v postgres
docker-compose up -d postgres
docker-compose exec api python scripts/init_db.py
```

### Worker not processing tasks

```bash
# Check Celery status
docker-compose exec worker celery -A app.tasks.celery_app inspect ping

# Check Redis connection
docker-compose exec redis redis-cli

# Restart worker
docker-compose restart worker
```

### Port already in use

```bash
# Change ports in .env
API_PORT=8001
WEB_PORT=3001

# Restart
docker-compose up -d
```

## 📚 Advanced Usage

### Development Mode (with hot reload)

```bash
docker-compose -f docker-compose.yml -f docker-compose.dev.yml up -d
```

Includes:
- Hot reload for Python/Node code
- pgAdmin (port 5050)
- Redis Commander (port 8081)
- Adminer (port 8080)

### Production Deployment

```bash
# Generate SSL certificates
mkdir -p ssl
openssl req -x509 -newkey rsa:4096 -keyout ssl/key.pem -out ssl/cert.pem -days 365 -nodes

# Start with Nginx proxy
docker-compose --profile prod up -d
```

### Run Tests

```bash
docker-compose run --rm api pytest tests/ -v --cov=app
```

### Database Backup/Restore

```bash
# Backup
docker-compose exec -T postgres pg_dump -U osint osint | gzip > backup.sql.gz

# Restore
gunzip < backup.sql.gz | docker-compose exec -T postgres psql -U osint osint
```

### Execute Migrations

```bash
docker-compose exec api alembic upgrade head
```

## 🔐 Security

### Change Default Passwords

```bash
# Generate strong password
openssl rand -base64 32

# Update in .env
DB_PASSWORD=new_strong_password
REDIS_PASSWORD=new_strong_password
JWT_SECRET_KEY=new_jwt_secret
```

### Enable HTTPS

Place certificates in `ssl/` directory and use `--profile prod`:

```bash
docker-compose --profile prod up -d
```

### Network Isolation

Services run on isolated network `osint-net`:

```bash
docker network inspect osint-platform-2026_osint-net
```

```bash
  aquasec/trivy image osint-api:latest
```

## 📈 Performance Tuning

### PostgreSQL

- Shared buffers: 256MB (configurable)
- Maintenance work mem: 64MB
- Effective cache: 1GB

Adjust in `docker-compose.yml`:

```yaml
postgres:
  environment:
    POSTGRES_INITDB_ARGS: "-c shared_buffers=512MB -c effective_cache_size=2GB"
```

### Redis

- Max memory: 512MB (with LRU eviction)
- Persistence: AOF enabled

### Celery

- Concurrency: 4 workers (default)
- Max tasks per child: 1000

Increase in `docker-compose.yml`:

```yaml
worker:
  command: celery -A app.tasks.celery_app worker --concurrency=8 --max-tasks-per-child=1000
```
